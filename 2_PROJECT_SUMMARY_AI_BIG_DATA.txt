PROJECT SUMMARY  |  AI + Big Data Final Project
Aspect-Based Sentiment Analysis + Review Quality Filter  |  Amazon US Electronics Dataset
1. Tujuan Proyek
Membangun sistem AI yang menganalisis ulasan pelanggan Amazon untuk menghasilkan keputusan otomatis apakah suatu produk layak Dipertahankan, Dievaluasi, atau Ditarik dari platform e-commerce.

Sistem ini menggabungkan dua pendekatan:
• Aspect-Based Sentiment Analysis (ABSA) - analisis sentimen bukan secara keseluruhan, tapi per aspek produk: harga, kualitas, pengiriman, dan layanan
• Review Quality Filter - menyaring review tidak reliable sebelum dianalisis agar keputusan lebih akurat

4. Detail Tiap Step
Step 1 - Preprocessing dengan Apache Spark
Tujuan: membersihkan dan menyiapkan data skala besar. Spark dipakai karena dataset 3M+ baris tidak efisien dengan Pandas biasa.
• Drop baris yang null pada kolom review_body, star_rating, helpful_votes
• Hapus karakter HTML dari teks review, ubah ke lowercase
• Buat kolom baru: quality_ratio = helpful_votes / (total_votes + 1)
• Buat kolom label sentimen dari star_rating (0=Negatif, 1=Netral, 2=Positif)
• Sample ~40% data untuk efisiensi proses selanjutnya, simpan ke Parquet

Step 2 - Review Quality Filter (Model 1)
Tujuan: memprediksi apakah sebuah review cukup reliabel untuk dijadikan dasar keputusan. Target: review dengan quality_ratio > 0.5 dianggap reliable.
Fitur input model:
FiturDeskripsireview_lengthPanjang karakter review_bodyword_countJumlah kata dalam reviewquality_ratiohelpful_votes / (total_votes + 1)is_verifiedDari kolom verified_purchaseis_vineDari kolom vineexclamation_countJumlah tanda seru dalam tekscaps_ratioRasio huruf kapital terhadap total karakter
• Algoritma utama: XGBoost Classifier
• Algoritma baseline/pembanding: Random Forest
• Metrik evaluasi: Accuracy, Precision, Recall, F1-Score, ROC-AUC

Step 3 - Aspect-Based Sentiment Analysis / ABSA (Model 2)
Tujuan: menganalisis sentimen bukan secara keseluruhan, tapi per aspek spesifik yang disebutkan dalam review.

3a. Aspect Extraction (rule-based keyword matching):
AspekContoh keyword yang dicariHargaexpensive, cheap, worth, overpriced, affordable, price, cost, valueKualitasbroken, durable, quality, material, build, sturdy, defective, excellentPengirimanshipping, delivery, arrived, package, fast, slow, damaged, lateLayanancustomer service, refund, return, support, warranty, replacement
Jika keyword aspek ditemukan dalam review -> aspek PRESENT, lanjut ke klasifikasi sentimen. Jika tidak ditemukan -> label aspek = N/A.

3b. Sentiment Classification per aspek:
• Algoritma: DistilBERT fine-tuned (model transformer ringan, lebih cepat dari BERT penuh)
• Input: kalimat dalam review yang mengandung keyword aspek
• Output per review: { harga: Pos/Neg/NA, kualitas: Pos/Neg/NA, pengiriman: Pos/Neg/NA, layanan: Pos/Neg/NA }
• Pretrained model HuggingFace yang dipakai: distilbert-base-uncased-finetuned-sst-2-english

Step 4 - Scoring Engine
Tujuan: mengagregasi semua hasil ABSA menjadi satu skor per produk.
aspect_score[X] = (jumlah review Positif aspek X) / (total review yang menyebut aspek X)
product_score   = rata-rata semua aspect_score x rata-rata quality_score
weakness_flag   = aspek dengan aspect_score terendah

Step 5 - Decision Engine
Tujuan: menghasilkan keputusan otomatis platform berdasarkan product_score.
KondisiKeputusanAksiproduct_score >= 0.65PERTAHANKANListing tetap aktif0.35 <= score < 0.65EVALUASINotifikasi seller: perbaiki aspek [weakness_flag]product_score < 0.35TARIKListing disuspend, review ulang 30 hari
Step 6 - Visualisasi (4 wajib)
VisualisasiIsiRadar ChartProfil 4 aspek sentimen per produk dalam satu grafik polarHeatmapAspect score vs product category, temukan pola masalah globalWordCloudKata paling sering muncul pada review negatif tiap aspekConfusion MatrixAkurasi Model 1 (Quality Filter) dan Model 2 (ABSA)
